简介

CAP定理指以下3条不可能同时满足：

  • 一致性：所有节点访问同一份最新的数据。
  • 可用性：数据或服务一直处于可用状态。
  • 分区容错性：系统或网络某部分出现故障时，整体仍然可以对外提供服务。

12-factor应用：

  • 一份代码，多次部署
  • 隔离依赖，显式声明
  • 配置存储在环境中
  • 应用无状态
  • 每个服务管理自己的数据
  • 并发
  • 快速启动，优雅退出
  • 把日志当做事件流
